# biaospec-plan — 把已批准的 spec 拆成可勾选的 todo

Claude 本来就会列 todo。这个 skill 只保证三件裸对话给不了的事：计划挂在**已批准的 spec** 名下、每条任务**回指验收标准**、产物**留在文件里**不随对话蒸发。其余怎么拆、怎么验证，用你的常识。

## 前置检查

优先用对话里已知的 spec 路径（brainstorm 刚移交的、或用户点名的）。不明确时去 `.biaospec/` 下找状态为"已批准"的 `spec.md`——别凭日期猜，多于一个就列候选（目录名 + 一句话目标）让用户指认。没有 spec、或设计仍含糊 → 先回 `biaospec-brainstorm`，不要凭空拆。

## 拆计划

把 todo 追加到 spec 的 `## 实施计划（Plan）` 节：

```markdown
## 实施计划（Plan）

- [ ] **Task 1：<名称>**
  - 文件：<精确路径，创建/修改哪些>
  - 内容：<做什么>
  - 对应：AC-1、AC-3
  - 验证：<怎么确认这个 task 完成>
- [ ] **Task 2：<名称>**
  - …
```

三条硬规则（其余交给常识）：

1. **每个 task 标 `对应：AC-x`**，落盘后逐条核对每个 AC 都被至少一个 task 覆盖，有缺口就补 task
2. **计划留在文件里**——执行中发现的坑追加到对应 task 下（`发现：…`），对话会被压缩，文件不会
3. **不留 placeholder**——别写 TBD / "同 Task N" / "适当处理错误"，说不清的直接写清

## 执行

获批后把状态行改"实施中"，按 todo 逐个做：完成一个勾一个、当场跑验证，别攒到最后。全做完把状态行改"完成"，逐条核对验收标准。发现 spec 有缺陷 → 回 `biaospec-brainstorm` 澄清，别自作主张偏离。
